# PoE Stash Helper

Desktop helper for **Path of Exile 1** that reads your stash through the
official API, scores items against your filters, and shows an overlay above
the stash grid.

> **Third-party notice**
> This product is a third-party fan tool. It is **not affiliated with or
> endorsed by Grinding Gear Games**.

---

## Features

| Feature | Details |
|---|---|
| Active mod filtering | Build slot-based filter sets and score items against the mods you care about |
| Overlay scoring | Draw coloured outlines and badges directly above matching stash items |
| Official authentication | Uses the official Path of Exile OAuth 2.1 Authorization Code + PKCE flow |
| Official stash access | Reads stash tabs through the official account API |
| Official trade stats | Loads stat definitions from the official trade stats endpoint |
| RePoE / PoB data rebuild | Rebuilds generated base-type and mod data from maintained public data sources |

## Setup

### 1. Register your application

1. Go to <https://www.pathofexile.com/developer/docs/authorization>
2. Create a new application using the public desktop client flow
3. Set redirect URI to `http://localhost:7878/callback`
4. Copy your **Client ID** into `config.py` → `OAUTH["client_id"]`
5. Set your contact information in the configured `User-Agent`

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

### 3. Run

```bash
python main.py
```

On first launch the app opens the browser for the official Path of Exile
login flow. After approval the token is cached locally.

## Usage

1. Open the **Account** tab and log in through the official OAuth flow
2. Load your stash tabs
3. Open **Scan & Filters** and build active mod groups for the slots you care about
4. Press **Apply & Scan**
5. Adjust **Overlay** settings if the grid needs calibration

## GGG and API compliance

This project is intentionally built around the documented Path of Exile API:

- official OAuth 2.1 Authorization Code + PKCE login flow
- official account stash endpoints
- official trade stats endpoint
- identifiable `User-Agent`
- conservative request pacing and retry handling
- no website scraping and no in-game automation
