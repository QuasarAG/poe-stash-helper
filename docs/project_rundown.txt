PoE Stash Helper — project rundown
===================================

This text file explains, in plain language, what each folder and file is for.
It is meant for a beginner maintainer who wants a quick map of the project.

[File] .gitignore
  Tells Git to ignore generated files such as caches, bytecode folders, and local token files.

[File] Inspect RePoE Cache.bat
  Windows helper that inspects the local RePoE cache used by rebuild tools.

[File] Launch PoE Stash Helper.bat
  Windows launcher for the desktop app with bytecode-cache creation disabled.

[File] README.md
  Main human-readable project introduction and setup guide.

[File] Update from RePoE.bat
  Windows launcher for rebuilding generated Path of Exile data from maintained public sources.

[Folder] api/
  HTTP-facing layer for official Path of Exile web requests. Keeps raw network code out of UI and business logic.

[File] api/__init__.py
  Package export for retrieving the shared API manager.

[Folder] api/cache/
  Small API-specific caches used to avoid unnecessary repeated requests.

[File] api/cache/__init__.py
  Marks the API cache folder as a Python package.

[File] api/cache/stash_response_cache.py
  Saves and restores cached stash responses and ETags.

[Folder] api/clients/
  Thin request clients. Each file focuses on one external HTTP area.

[File] api/clients/__init__.py
  Package marker for API clients.

[File] api/clients/oauth_client.py
  Official OAuth 2.1 Authorization Code + PKCE desktop client for Path of Exile.

[File] api/clients/stash_client.py
  Thin official stash-endpoint client.

[File] api/clients/stats_client.py
  Thin official trade-stats endpoint client plus disk-cache handling.

[File] api/manager.py
  Creates and owns the shared API clients used by the app.

[Folder] api/policies/
  Shared request rules such as headers, rate limiting, and friendly error parsing.

[File] api/policies/__init__.py
  Package marker for shared request-policy helpers.

[File] api/policies/error_policy.py
  Friendly API error parsing and Retry-After handling.

[File] api/policies/rate_limiter.py
  Conservative request pacing and response-header based throttling.

[File] api/policies/request_policy.py
  Shared request header builder, including User-Agent and Authorization headers.

[File] config.py
  Central configuration values such as API settings, OAuth settings, and default UI behaviour.

[Folder] controllers/
  Qt application orchestration. Connects views, workers, services, and overlay behaviour.

[File] controllers/__init__.py
  Package marker for controllers.

[File] controllers/app_controller.py
  Main application controller. Connects the window, overlay, workers, tray, and hotkeys.

[Folder] data/
  Generated database files and user-owned saved data.

[File] data/__init__.py
  Package marker for the data folder.

[File] data/base_types.json
  Generated base-item database used by the base filter and item parser.

[File] data/config.json
  User configuration saved between runs.

[File] data/loadouts.json
  User-created loadouts and filter definitions.

[File] data/mod_data.py
  Generated modifier database used for mod search and scoring.

[Folder] docs/
  Plain-language documentation meant for a beginner maintainer.

[Folder] logic/
  Pure project logic: item parsing, filtering, scoring, and related helper rules.

[File] logic/__init__.py
  Package marker for pure logic modules.

[File] logic/item_filter.py
  Helpers for matching parsed items against built filters.

[File] logic/item_parser.py
  Turns raw stash JSON into parsed item objects.

[File] logic/mod_query.py
  Search and query helpers over generated modifier data.

[File] logic/mod_scorer.py
  Weighted modifier scoring logic.

[File] logic/tier_utils.py
  Helpers for modifier tier and value-range display.

[File] logic/unified_filter.py
  Applies slot-aware active mod and property filters to parsed items.

[File] logic/window_finder.py
  Detects and tracks the Path of Exile window for overlay behaviour.

[File] main.py
  Thin application entry point that creates the Qt app and starts the controller.

[Folder] models/
  Small dataclasses that give important app data a clear shape.

[File] models/__init__.py
  Package export for the small shared dataclasses.

[File] models/scan_models.py
  Dataclasses for one scan request and one scan result.

[File] models/stash_models.py
  Dataclass for one stash-tab summary shown in the UI.

[File] models/trade_stat_models.py
  Dataclass for one trade-stat entry from the official stats endpoint.

[Folder] repositories/
  File-backed data access. Reads and writes JSON or generated data for the rest of the app.

[File] repositories/__init__.py
  Package export for repository helpers.

[File] repositories/base_repository.py
  Single source of truth for generated base-type data.

[File] repositories/config_repository.py
  Read and write user config values.

[File] repositories/loadout_repository.py
  Read and write saved loadouts.

[File] requirements.txt
  Python dependency list for the project.

[Folder] services/
  Reusable workflows that sit above repositories and API clients.

[File] services/__init__.py
  Package export for common service entry points.

[File] services/data_update_service.py
  Shared rebuild workflow used by both UI and CLI tools.

[File] services/oauth_login_service.py
  High-level login workflow wrapper around the OAuth client.

[File] services/scan_payload_service.py
  Builds scan payload data from the current tab state.

[File] services/stash_service.py
  Higher-level stash workflow and stash-tab normalization helpers.

[File] services/stats_service.py
  Small service wrapper around the official trade-stats client.

[Folder] tools/
  Command-line maintenance scripts, mainly for rebuilding generated data.

[File] tools/__init__.py
  Package marker for command-line tools.

[File] tools/build_base_db.py
  Builds generated base-type data from RePoE base item data.

[File] tools/build_mod_db.py
  Builds generated modifier data from PoB / RePoE-style sources.

[Folder] tools/common/
  Shared helpers used by multiple command-line tools.

[File] tools/common/__init__.py
  Package marker for shared tool helpers.

[File] tools/common/http_cache.py
  Shared JSON download and local-cache helper used by rebuild tools.

[File] tools/inspect_cache.py
  CLI helper for inspecting cached rebuild inputs.

[File] tools/update_all.py
  CLI entry point for rebuilding base data, mod data, or both.
